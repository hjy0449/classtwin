# ClassTwin AI

교사를 위한 AI 기반 반배정·자리배정 도우미. 학생 데이터를 엑셀로 업로드하면, 갈등 관계·시력·특수교육·성적·키 같은 조건을 종합해 최적 자리를 자동으로 배치해주는 웹 앱입니다.

---

## 폴더 구조

```
classroom/
├── README.md
├── backend/

│   ├── .env.example         # 환경변수 템플릿

    └── student_template_자리배정.xlsx    # 자리배정용 학생 데이터 템플릿
```

## 주요 기능 한눈에

**반배정 (ban 플로우)** — 학생을 여러 반으로 나누기
- 갈등 페어 분리, 성비·성적 균등 분산, 고립 학생 감지
- **AI 챗봇**으로 자연어 조건 입력 ("민준이랑 서연이 분리해줘")
- 교사 소견(자유 텍스트)을 LLM이 파싱해 리더십·사교성·집중력·긍정성 4가지 특성 추출
- Simulated Annealing 기반 최적화로 3가지 배치안 생성

**자리배정 (seat 플로우)** — 한 반 안에서 좌석 배치
- 시력배려·특수교육·ADHD 학생 앞자리 우선
- 갈등 페어는 인접하지 않게 자동 분리
- 키순 정렬 (앞줄 작음 → 뒷줄 큼)
- **드래그로 균형조건 우선순위 조절** — 무엇을 더 중요하게 볼지 사용자가 결정
- **추가고려사항** 자유 입력 ("정민우 앞자리로", "한자영과 유선우 옆자리로") — LLM 파싱 + 규칙 기반 fallback
- 수동 swap 조정 모드, 통계·알림 즉시 재계산
- AI 문서 자동 생성 (교사용 결과표, 학생별 배정 이유 검색, 형평성 리포트)

## 설치 및 실행

### 필요 환경
- Python 3.10 이상 (개발은 3.13 환경)
- 최신 웹 브라우저 (Chrome, Edge 등)

### 1. 패키지 설치

```powershell
cd backend
pip install -r requirements.txt
```

`requirements.txt`에 포함된 패키지 (pinned 버전):

```
fastapi==0.111.0
uvicorn==0.30.1
httpx==0.27.0
python-dotenv==1.0.1
pydantic==2.7.1
pandas==2.2.2
openpyxl==3.1.2
python-multipart==0.0.9
```

### 2. `.env` 파일 생성

`.env.example`을 복사해서 `.env`로 만들고 실제 OpenRouter API 키를 입력하세요:

```powershell
cd backend
Copy-Item .env.example .env
# .env 파일 열어서 OPENROUTER_API_KEY=sk-or-... 입력
```

OpenRouter 키는 [openrouter.ai](https://openrouter.ai)에서 무료로 발급받을 수 있습니다. 키 없이도 앱은 동작하며, 이 경우 LLM이 필요한 기능(교사소견 분석, 챗봇, 추가고려사항 파싱, AI 문서 생성)은 규칙 기반 fallback으로 대체됩니다.

### 3. 백엔드 실행

```powershell
cd backend
uvicorn main:app --port 8000
```

콘솔에 `INFO: Application startup complete.`가 뜨면 성공. `http://localhost:8000`에서 동작하며, `http://localhost:8000/docs`에서 Swagger UI로 전체 API를 확인·테스트할 수 있습니다.

> **참고**: `--reload` 옵션은 권장하지 않습니다. Windows + WatchFiles 조합에서 출력 버퍼링 이슈로 `print()` 디버그 로그가 콘솔에 안 보이는 경우가 있습니다. 코드 수정 후엔 Ctrl+C → 다시 실행으로 재시작하세요.

### 4. 프론트엔드 실행

새 PowerShell 창에서:

```powershell
cd frontend
python -m http.server 5500
```

브라우저로 `http://localhost:5500` 접속.

### 5. (선택) 회귀 테스트 실행

알고리즘 수정 후 기능이 안 깨졌는지 확인하려면:

```powershell
cd backend
pip install pytest pytest-asyncio
python -m pytest test_seat_e2e.py -q
```

자리배정 생성·평가·문서·이력 4개 테스트가 통과해야 정상입니다.

## 사용 흐름

### 학생 데이터 준비

`frontend/` 폴더에 두 가지 엑셀 템플릿이 있습니다. 용도에 맞는 템플릿을 열어 학생 정보를 입력하세요:

- **`student_template_반배정.xlsx`** — 반배정용 (학년 전체 학생 기준)
- **`student_template_자리배정.xlsx`** — 자리배정용 (한 반 기준)

두 템플릿 모두 시트 세 개로 구성됩니다:

- **학생목록**: 이름, 성별, 성적(상/중/하), 키(cm), 시력(정상/약함), 주의력(높음/중간/낮음), 특수교육(ADHD/학습장애/없음), 교사소견
- **관계데이터**: 학생A, 학생B, 관계유형(갈등/친함), 심각도, 메모
- **사용안내**: 작성 방법 안내 (수정할 필요 없음)

입력 예시 행(`예) 김민준` 등)은 시스템이 자동으로 걸러냅니다.

### 반배정 플로우

1. **ban1 (학생 데이터 업로드)**
   - 엑셀 업로드 → 자동으로 교사소견 AI 분석 시작 (백그라운드)
   - 학생 통계 확인 (총 인원, 성비, 평균 성적 등)
   - 파싱 완료 후 다음 단계로 이동 가능

2. **ban2 (조건 설정)**
   - **AI 챗봇으로 자연어 조건 입력**
     - 예: "외향적인 학생들 각 반에 골고루 배치해줘"
     - 예: "김민준이랑 이서연은 분리해줘"
   - 지원 조건 타입:
     - 절대 조건: 특정 학생 분리/같은 반
     - 형제자매 분리
     - 전학생 배려
     - 리더 분산 (리더십/외향성/집중력/긍정성 특성별)

3. **ban3 (AI 분석 실행)**
   - 교사소견 AI 분석 완료 대기
   - "AI 분석 실행" 버튼으로 3개 배치안 생성

4. **ban4 (배치안 확정)**
   - 3개 배치안 비교 (학급 안정성 지수)
   - AI 추천 배치안 선택 또는 수동 선택

5. **ban5 (문서 생성)**
   - 교사용 배치 결과표
   - 학생별 배정 이유
   - 학부모 민원 대응 문서

### 자리배정 플로우

1. 홈에서 "자리배정" 선택
2. 학생 데이터 엑셀 업로드 — 관계 데이터까지 한 파일에서 같이 읽어옵니다
3. **seat1 (조건 설정)**
   - 교실 행/열 수 입력
   - 절대조건은 학생 데이터에서 자동 도출됩니다 (시력약함, ADHD 등)
   - **균형조건 우선순위 드래그** — 위에 둘수록 강하게 반영. 5개 항목:
     - 시력배려 — 앞자리 우선
     - 특수교육(ADHD/학습장애) — 앞자리 우선
     - 키 순서 배치 (앞 낮음→뒤 높음)
     - 성별 교차 배치
     - 친한 친구 분리
   - **추가고려사항** textarea에 자연어로 원하는 조건 입력 (선택)
     - 예: `박지호 앞자리로`, `한자영과 유선우 분리`, `정민우 이승호 옆자리로`, `신동현 교사 근처`
4. **seat2 (자동 배치 결과)** — AI가 최적 자리를 계산해 보여줌. 상단에 추가고려사항 해석 결과가 파란 notice로 표시됨
5. **seat3 (수동 조정)** — 두 자리를 클릭하면 서로 swap. 통계·알림이 즉시 재계산됨
6. **seat4 (최종 확정)** — 최종 검토
7. **seat5 (문서)** — AI가 생성한 교사용 결과표 + 형평성 리포트 + 학생별 배정 이유 검색

### 자리 분류 표시

| 학생 분류 | 배경색 | 아이콘 | 테두리 |
|---|---|---|---|
| 일반 | 흰색 | — | 기본 |
| 시력배려 | 파란색 | — | 기본 |
| 특수교육(ADHD/학습장애) | 흰색 | ⚡ | 기본 |
| 시력배려 + 특수교육 | 파란색 | ⚡ | 기본 |
| 갈등주의 (어떤 조합이든) | 위 그대로 | 위 그대로 | **빨강** |

갈등주의는 "이 학생은 갈등 페어 이력이 있다"는 표시로, 인접 여부와 무관하게 갈등 페어에 등장하는 모든 학생에게 빨간 테두리가 붙습니다.

## 교사소견 AI 분석

교사소견 자유 텍스트를 LLM이 파싱해 4가지 특성을 1~10점으로 자동 평가합니다:

- **leadership (리더십)** — "반장 기질", "리더형", "주도적"
- **sociability (사교성)** — "외향적", "친화력 좋음", "친구가 많음"
- **attention (집중력)** — "차분함", "성실함", "집중력 높음"
- **positive_influence (긍정성)** — "긍정적", "밝음", "분위기 메이커"

각 특성은 7점 이상이면 "특성 보유"로 판단되어 반배정 시 각 반에 고르게 분산됩니다.

**예시:**
```
교사소견: "반장 경험 있음. 친구들을 잘 이끌고 책임감이 강함."
→ AI 분석: leadership=9, sociability=7, positive_influence=8
```

## AI 챗봇 사용 가이드 (반배정)

### 질문만 하는 경우 (조건 추가 안 됨)

- "외향적인 학생 누구있지?"
- "리더십 좋은 애들 알려줘"
- "차분한 친구 누가 있어?"

### 조건으로 인식되는 요청

- "외향적인 학생들 **분산해줘**" → trait_balance
- "김민준이랑 이서연 **분리해줘**" → 분리 조건
- "박지호랑 남궁현 **같은 반으로 해줘**" → 같은 반 조건
- "리더십 학생들 **골고루 배치해줘**" → trait_balance

**핵심 규칙:** `~해줘`, `~배치해`, `~분산해` 같은 **행동 요청 동사**가 있어야 조건으로 추가됩니다.

### 챗봇 정확도 향상 (하이브리드 필터링)

AI 모델(GPT-4o-mini)과 규칙 기반 필터링을 조합해 정확도를 높였습니다. 질문 키워드가 감지되면 자동으로 `simple_question`으로 분류하여 조건 추가를 방지합니다:

```python
question_keywords = ["누가", "누구", "이름", "알려", "말해", "있지", "있어", ...]
```

**현재 정확도:** 약 85~90%

## 알고리즘 개요

### 반배정 — Simulated Annealing

**점수 함수에 포함되는 항목:**
- 갈등 페어 분리: 같은 반이면 `-100`
- 성비 균등: 각 반 성비 차이 최소화
- 성적 균등: 상/중/하 학생 골고루 배분
- 특성 균등 (trait_balance): 리더십/사교성/집중력/긍정성 각 반에 분산
- 친한 친구 같은 반: `+보너스`
- 고립 학생 보호: 친한 친구와 같은 반 배치

**3가지 배치안:**
- **배치안 A** — 전체 균형 최적화 (AI 추천)
- **배치안 B** — 친한 친구 같은 반 우선
- **배치안 C** — 성적/성비 균등 우선

### 자리배정 — `_score_seats` 점수 함수

각 자리 배치 후보에 대해 다음 항목을 합산해 총점을 매기고, SA가 2,000 iterations 동안 최대화합니다.

**고정 페널티/보너스 (가중치 무관)**
- 갈등 페어 인접: `-100`
- 런타임 `forced_front` (추가고려사항 앞자리 요청): `-60`
- 런타임 `forced_teacher_near`: `-50`
- 런타임 `forced_adjacent` (옆자리 요청):
  - 같은 행, 열 차이 1 → `+250`
  - 같은 행, 열 차이 2 → `+50`
  - 그 외 → `-40 × col_diff - 60 × row_diff`

**가중치 페널티 (균형조건 우선순위 반영)**
- 시력약함 3행 이상: `-50 × (r-1) × bw["vision"]` — 뒤로 갈수록 급격히 커짐
- 특수교육 3행 이상: `-50 × (r-1) × bw["special"]`
- 키 차이: `|실제 키 - 이상 키| × 1.0 × bw["height"]`
- 같은 성별 좌우 인접: `-3 × bw["gender_alt"]`
- 친한 친구 인접: `±5 × bw["friend_separate"]` (부호는 가중치 기준)

**가중치 값** (드래그 순위 → 곱셈 계수)
```
1순위 = 2.5    (최상위, 확실히 우선)
2순위 = 1.5
3순위 = 1.0    (기본)
4순위 = 0.7
5순위 = 0.5
```

1순위와 2순위 차이를 의도적으로 크게(0.5 → 1.0) 벌려서, 사용자가 1순위로 둔 조건이 확실히 나머지를 이기도록 설계했습니다.

### "밀리더라도 3행까지" 로직

시력약함·특수교육 학생이 앞 2줄에 다 못 들어가는 상황(예: 시력약함 11명 + 특수 4명 > 12석)에서는, 뒤로 갈수록 페널티가 선형 증가하도록 설계했습니다. 이로써 SA가 4·5행보다 3행을 선호하게 유도되어, 밀려나더라도 최대한 앞쪽에 배치됩니다.

### 추가고려사항 파싱 (이중 구조)

`extra_note` 텍스트는 두 단계로 처리:

1. **LLM 파싱** (`ai.parse_conditions`) — GPT-4o에게 자연어 → JSON 구조화
2. **Fallback 파서** (`_simple_parse_extra_note`) — LLM이 실패하거나 빈 결과를 반환하면 규칙 기반 파서가 대신 동작. 학생 이름 + 키워드(`앞자리`, `분리`, `옆자리`, `교사 근처`) 매칭으로 조건 추출

덕분에 LLM 키가 없거나 장애가 있어도 추가고려사항은 항상 동작합니다.

## 데이터 저장 방식

현재 **in-memory 딕셔너리**를 사용합니다 (`backend/database.py`). 서버를 재시작하면 모든 데이터가 초기화됩니다. 실사용 시 학생 데이터를 매번 엑셀로 다시 업로드해야 합니다.

`database.py`의 메서드 인터페이스는 SQLite·PostgreSQL·Supabase 등 실제 DB로 교체할 수 있도록 정리돼 있어, 영구 저장이 필요해지면 최소한의 수정으로 확장 가능합니다.

## 알려진 한계

- **in-memory 저장소** — 서버 재시작 시 데이터 손실. 개발·시연용으로는 충분
- **좌석 수 부족 시** — 시력배려 + 특수교육 학생 수가 앞 2줄 좌석 수보다 많으면, 일부는 수학적으로 밀려날 수밖에 없습니다. 이 경우 알고리즘은 "밀리더라도 3행까지"로 최대한 앞쪽에 배치합니다
- **LLM 의존도** — 교사소견 분석, 챗봇, AI 문서 생성은 LLM 키가 있어야 품질이 좋습니다. 키 없이도 동작은 하지만 fallback 수준의 텍스트만 나옵니다
- **브라우저 호환성** — 최신 Chromium 기반 브라우저(Chrome, Edge)를 전제로 개발되었습니다

## 트러블슈팅

**자리배정 결과에 균형조건이 반영이 안 되는 것 같음**
→ 백엔드 PowerShell 콘솔을 확인하세요. `⚖️ balance 수신: [...]`와 `⚖️ 계산된 가중치: {...}` 두 줄이 떠야 정상입니다. 안 뜨면 브라우저 캐시(Ctrl+Shift+R) 또는 백엔드 재시작이 필요합니다.

**추가고려사항이 반영 안 됨**
→ 콘솔에서 `📝 자리배정 요청 도착: extra_note=...`를 확인하세요. 빈 문자열이면 프론트의 textarea 입력이 수집되지 않은 것. 정상 문자열이면 그 아래 `✅ LLM 파싱` 또는 `🔧 fallback 파싱` 줄이 이어서 떠야 합니다.

**`.env`의 API 키가 인식 안 됨**
→ `backend/.env` 파일이 정확히 `backend/` 폴더 안에 있는지 확인하세요. uvicorn을 실행한 작업 디렉토리가 `backend/`여야 `load_dotenv()`가 찾을 수 있습니다. 프로젝트 루트에서 실행하면 인식 안 될 수 있으니 `cd backend` 후 실행하세요.

**8000 포트가 이미 사용 중 에러**
→ 이전 uvicorn 프로세스가 좀비로 남아 있을 수 있습니다:
```powershell
netstat -ano | findstr :8000
taskkill /F /PID <표시된 PID>
```
또는 간단히 컴퓨터 재부팅.

**localhost:5500이 안 열림 / 빈 화면**
→ 프론트 서버가 안 떠 있는 상태입니다. 새 PowerShell 창에서 `cd frontend; python -m http.server 5500`을 실행하세요. 백엔드 서버(8000)와 프론트 서버(5500)는 **별개 PowerShell 창에서 동시에** 떠 있어야 합니다.

**수동 swap 후 빨간 테두리가 사라짐**
→ 해결됨. swap 시 백엔드가 새 grid의 flags를 다시 계산해서 돌려줍니다.
